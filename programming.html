Decision structures

if/else

  • if: If true, executes the block of code.
  • else: If not, does not execute the block of code.
if (condition) {
  console.log("Condition is true.");
} else {
  console.log("Condition is false.");
}

Switch/case

  • switch: Executes a block of code depending on the value of the expression.
  • case: Defines the block of code to be executed for a specific value.
  • default: Defines the block of code to be executed if no case matches.
int value = 2;

switch (value) {
    case 1:
        /* block of instructions 1 */
        break;
    case 2:
        /* block of instructions 2 */
        break;
    case 3:
        /* block of instructions 3 */
        break;
    default:
        /* default block of instructions */
        break;
}

Repetition structures

while loop

  • while: Executes a block of code as long as a specified condition is true.
int i = 0;

while (i < 10) {
    /* block of instructions */
    i++;
}

do-while loop

  • do-while: Executes a block of code once, and then repeats the loop as long as a specified condition is true.
int i = 0;

do {
    /* block of instructions */
    i++;
} while (i <= 0);

for loop

  • for: Executes a block of code a specified number of times.
for (int i=0; i<5; i++) {
    /* block of instructions */
}

Structure of a Class

A class in object-oriented programming is a generic abstraction of objects, representing their instances. Its basic structure includes the class name, attributes, and methods.

Guidelines and best practices

  • Class names start with an uppercase letter. For names with more than one word, use UpperCamelCase.
  • Attributes and methods start with a lowercase letter. For names with more than one word, use lowerCamelCase.
  • Attributes use access modifiers like private to adhere to encapsulation principles, as discussed in the next section.

Encapsulation

Encapsulation in object-oriented programming hides internal class details and exposes only necessary interfaces for interaction.

It ensures data integrity and reduces dependency on implementation details by declaring attributes as private, thus controlling access and enforcing modular design principles. This approach enhances code reliability, security, and maintainability.

  • public: A member (attribute or method) marked as public in programming can be accessed from anywhere, including within the same class, classes in the same package, and classes in different packages.
  • protected: The protected modifier allows a member to be accessed within its own class, by classes in the same package, and by subclasses (even if they are in different packages).
  • private:A private member can only be accessed within its own class. It cannot be accessed from outside the class, not even by classes in the same package or subclasses.

Abstract classes and methods

An abstract class is one that cannot be instantiated.

  • This means that it is not possible to create objects of this type.
  • It is possible to create variables or references of this type.

An example is the Animals class, which is a generalization of all animals, so it only exists in the world of ideas and does not need to be instantiated.

Abstract methods:

  • Cannot have a method body.
  • Must be declared in an abstract class
  • Abstract methods must be overridden by a subclass, ensuring programmers implement and redefine them. Abstract classes typically contain abstract methods, partially implemented methods, or fully implemented methods.

Polymorphism

Polymorphism, meaning "many forms," refers to the ability of a variable or reference to represent different types of objects. In programming, polymorphism allows variables to refer to multiple different types of objects, thereby exhibiting various forms or behaviors depending on the context.

  • Polymorphism in Java allows a variable of a superclass type (`Shape`) to refer to objects of its subclasses (`Rectangle`, `Square`, `Ellipse`, `Circle`). This flexibility enables the use of common methods and behaviors defined in the superclass across different types of objects, enhancing code reusability and flexibility in object-oriented programming.
  • Example:
Shape shape = new Rectangle();
shape = new Square();
shape = new Ellipse();
shape = new Circle();
